# دليل تجهيز بيانات العرض

إذا كان الهدف الآن هو تجهيز النظام للعرض أمام الإدارة، فوجود بيانات حقيقية أو شبه حقيقية للمدارس سيجعل العرض أقوى بكثير من بيانات الـ demo العامة.

## هل أحتاج ملفات بيانات المدارس؟

نعم، يفضّل ذلك جدًا.

بدون هذه الملفات يمكننا العرض ببيانات تجريبية، لكن مع بيانات المدارس الفعلية يصبح العرض:

- أقرب للواقع
- أوضح للإدارة
- أسهل في تقييم جاهزية النظام
- مناسبًا لاحقًا للتحويل إلى استيراد فعلي داخل النظام

## ما الامتداد الأفضل؟

الترتيب الموصى به:

1. `xlsx`
2. `csv`
3. `json`

## التوصية العملية

- إذا كانت الإدارة أو فريق المدارس سيملأ البيانات يدويًا:
  - استخدم `xlsx`
- إذا كنت تريد أن أتعامل معها بسرعة في النظام أو أحولها لاحقًا إلى seed/import:
  - استخدم `csv` بترميز `UTF-8`

## ما الملفات المطلوبة؟

الحد الأدنى الموصى به:

1. ملف المدارس
2. ملف المستخدمين
3. ملف الطلاب

والأفضل لاحقًا:

4. ملف أولياء الأمور
5. ملف المعلمين
6. ملف ربط الطلاب بأولياء الأمور

## أسماء القوالب الجاهزة داخل المستودع

- [users_template.csv](./import-templates/users_template.csv)

## أهم الملاحظات

- استخدم نفس `school_code` في كل الملفات لربط البيانات ببعضها.
- احفظ ملفات `csv` بترميز `UTF-8`.
- يفضّل عدم وضع تنسيقات Excel معقدة أو دمج خلايا.
- التواريخ يفضّل أن تكون بصيغة `YYYY-MM-DD`.
- القيم المنطقية يفضّل أن تكون: `true/false` أو `1/0`.

  - `students`

أو بدلًا من ذلك:

- 3 ملفات منفصلة `csv`

## ماذا سأفعل بعد استلامها؟

بعد استلام الملفات أستطيع:

1. مراجعة الأعمدة وتنظيفها
2. مطابقة البيانات مع هيكل النظام الحالي
3. تحويلها إلى seed data أو import-ready files
4. تجهيز نسخة عرض خاصة بالإدارة ببيانات حقيقية
