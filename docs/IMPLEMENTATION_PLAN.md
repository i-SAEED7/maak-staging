## مخرجات مرحلة 0

- إنشاء البنية الأساسية للمجلدات.
- تجهيز ملفات البيئة.
- تجهيز `docker-compose.yml` و`docker-compose.prod.yml`.
- إعداد ملفات أولية لـ Nginx وPHP وPostgreSQL.
- توثيق نقطة البداية للمطورين.

## مخرجات التحضير التنفيذي بعد مرحلة 0

- تصميم قاعدة البيانات الأولية كاملة على مستوى التوثيق وSQL.
- كتابة مخطط API واضح ومقسم حسب الوحدات.
- إنشاء هيكل backend وfrontend كمسارات وملفات placeholder.
- إضافة ملفات توثيق للهندسة الخلفية والواجهة وأولويات التنفيذ التالية.
- إضافة مصفوفة صلاحيات تفصيلية قابلة للتحويل إلى Seeders.
- إضافة عقود request/response للعمليات الأساسية.
- إضافة خرائط Routes واضحة للخلفية والواجهة.
- إضافة بيانات مرجعية أولية وملفات Seed بصيغ JSON وSQL.
- إضافة وثائق workflow وvalidation وtest strategy وADR.

## بوابة الانتقال إلى مرحلة 1

لا تبدأ مرحلة 1 إلا بعد توفر الأدوات التالية في البيئة:

- `composer`
- `php`
- `node`
- `npm`
- `docker`

وعند توفرها يكون التسلسل الفوري:

1. توليد مشروع Laravel داخل `backend/`
2. توليد مشروع React + Vite داخل `frontend/`
3. تثبيت الحزم الأساسية حسب وثيقة المواصفات
4. تفعيل المصادقة والصلاحيات والعزل
