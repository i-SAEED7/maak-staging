# حالة المشروع الحالية

## المنجز داخل المستودع

- وثيقة مواصفات أساسية
- هيكل مشروع عام
- إعدادات بيئة أولية
- مشروع Laravel فعلي داخل `backend`
- `migrations` و`seeders` منفذة فعليًا وتعمل داخل Laravel
- تصميم قاعدة البيانات
- SQL أولي
- مواصفات API
- عقود request/response
- مصفوفة صلاحيات
- خرائط routes
- بيانات مرجعية أولية
- Seeds وplaceholders للطبقات الأساسية
- وثائق workflows وvalidation وtest strategy وADR
- Blueprints تنفيذية للوحدات الأساسية
- مواصفات شاشات الواجهة
- خطة migrations مفصلة على دفعات
- DTOs وEnums وTraits وSupport placeholders
- Migration stubs أولية للكيانات الحرجة
- هياكل methods للـ controllers والخدمات الأساسية
- خرائط تكامل بين الشاشات والـ endpoints
- قواعد أولية لـ requests/resources/policies
- Migration stubs إضافية للعلاقات والجداول التابعة
- تغطية أولية لوحدات portfolios وstudent reports وreports والاتصالات
- توسعة صفحات الواجهة والخدمات لتغطي معظم المسارات الموثقة
- ربط أولي بين الأدوار والصلاحيات على مستوى seed-data
- hooks وstores تمهيدية للإشعارات والرسائل والتقارير
- طبقة models/events/listeners/observers التمهيدية أصبحت موجودة
- خرائط الحالة والعلاقات والمراقبة أصبحت موثقة
- دعم Requests المساندة والتحكم بالصلاحيات وUAT صار موثقًا وممهدًا
- Blueprint للاختبارات وCI وOpenAPI أصبح موجودًا
- onboarding والبيئة ومزودو الخدمة صاروا موثقين ومجهزين
- تفعيل `tenant context` فعليًا داخل Laravel
- تنفيذ CRUD فعلي لوحدات `schools`, `users`, `students`
- تنفيذ فعلي لوحدة `IEP` مع المسار التشغيلي الأساسي والإصدارات والتعليقات وبيانات `PDF`
- تنفيذ فعلي لوحدة `Messages` مع المحادثات والسلاسل والإرسال وتعليم الرسائل كمقروءة
- تنفيذ فعلي لوحدة `Notifications` مع عرض التنبيهات وتعليم المفرد والجميع كمقروء
- تنفيذ فعلي لوحدة `Reports` مع تقارير المدرسة والطالب والمقارنة وPivot ومعاينة التصدير
- تنفيذ فعلي لوحدة `Supervision` مع الزيارات الإشرافية وإنهاء الزيارة والتوصيات
- تنفيذ فعلي لوحدة `Files` مع الرفع والاستعراض والرابط المؤقت والحذف المنطقي
- تفعيل `policies` وربطها بالـ controllers في الوحدات المنفذة
- suite اختبارات Laravel أصبحت صالحة للتشغيل حاليًا مع نتائج: `3 passed, 10 skipped`
- Browser Demo فعلي موجود على `/` مع حسابات Demo وعمليات مباشرة من المتصفح تشمل المدارس والمستخدمين والطلاب و`IEP` و`Messages` و`Notifications` و`Reports` و`Files`
- مشروع React/Vite فعلي موجود الآن داخل `frontend/`
- `frontend` تم التحقق من بنائه بنجاح عبر `npm run build`
- تم بناء بوابة عامة `Portal` مستقلة فوق النظام الحالي بدون كسر الوحدات الداخلية
- تم فصل التنقل إلى مسارين واضحين:
  - البوابة العامة على `/`
  - النظام الداخلي للمدارس والإدارة على `/app`
- تم إضافة صفحات البوابة العامة: `الرئيسية`, `عن القسم`, `الخدمات`, `البرامج`, `الإحصائيات`, `الخريطة التفاعلية`, `تواصل معنا`, `تسجيل الدخول`
- تم إضافة صفحات انتقالية بعد تسجيل الدخول:
  - `/select-school` للمشرف متعدد المدارس
  - `/schools/:schoolId` كبوابة مدرسة قبل الدخول إلى وحدات النظام
- تم إضافة `Portal API` عامة للبرامج والمدارس والإحصائيات عبر:
  - `GET /api/portal/programs`
  - `GET /api/portal/schools`
  - `GET /api/portal/statistics`
- تم تطبيق منطق توجيه بعد تسجيل الدخول حسب نوع الارتباط بالمدارس بدون إظهار الأدوار في واجهة تسجيل الدخول
- تم تنفيذ طبقة دخول جديدة فوق النظام الحالي بدون كسر المسارات القديمة:
  - `POST /api/auth/school-login`
  - `POST /api/auth/central-login`
- تم إضافة حقول وهوية دخول جديدة:
  - `schools.school_code`
  - `schools.slug`
  - `users.username`
  - `users.is_central`
- تم ربط الدخول المدرسي بالتحقق الثلاثي:
  - بيانات اعتماد صحيحة
  - `school_code` صحيح
  - ارتباط فعلي بين المستخدم والمدرسة عبر جدول الإسناد
- تم تفعيل تتبع محاولات الدخول والإيقاف المؤقت بعد التكرار الفاشل

## غير المنجز بعد

- تحويل اختبارات الـ blueprint الحالية إلى اختبارات تكامل ووحدة حقيقية
- تفعيل مزيد من middleware/security/audit flows بشكل كامل
- ربط `Supervision` بواجهة Browser Demo أو بالواجهة React
- توسيع صفحات React لتشمل بقية الوحدات التشغيلية الأعمق
- تعميق تجربة صفحات المدرسة العامة لاحقًا عندما يبدأ تنفيذ قوالب مواقع المدارس

## المانع الوحيد الحالي

- لا يوجد مانع أدوات حاليًا؛ الأدوات الأساسية أصبحت متوفرة (`php`, `composer`, `node`, `npm`, `git`, `docker`)
- قد نحتاج لاحقًا فقط لتفعيل بعض امتدادات PHP الاختيارية مثل `zip`, `intl`, `gd`, `pdo_pgsql`, `redis` حسب اتجاه التشغيل النهائي
