# قواعد العزل متعدد المدارس

## المبادئ

- كل مستخدم يعمل ضمن سياق `school_id` أو مجموعة مدارس مسندة.
- `super_admin` لا يخضع لعزل مدرسة واحد.
- `supervisor` يخضع لقائمة مدارس مسندة عبر `user_school_assignments`.
- `principal` و`teacher` يعملان داخل مدرسة واحدة غالبًا.
- `parent` لا يرتبط بالمدرسة مباشرة بقدر ما يرتبط بأبنائه.

## قواعد الوصول

### `super_admin`
- يرى كل المدارس وكل البيانات.

### `admin`
- يرى البيانات المسموحة له حسب الإعداد المركزي.

### `supervisor`
- يرى المدارس المسندة إليه فقط.
- لا يرى طلاب مدارس خارج نطاقه.

### `principal`
- يرى فقط مدرسته.
- لا ينشئ أو يعدل بيانات خارجها.

### `teacher`
- يرى فقط الطلاب المسندين إليه، حتى داخل نفس المدرسة.

### `parent`
- لا يصل إلى أي طالب إلا عبر `student_guardians`.

## آليات التنفيذ لاحقًا

1. Middleware يحدد `tenant context`
2. Trait أو Scope على النماذج المرتبطة بالمدرسة
3. Policies للتأكد من الملكية أو الإسناد
4. فلاتر استعلام موحدة في طبقة الخدمة
