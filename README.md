# البوابة المتكاملة لقسم ذوي الإعاقة

هذا المستودع هو نقطة البداية التنفيذية للمشروع الموصوف في [maak.md](./maak.md).

## الحالة الحالية

- تم تجهيز الهيكل الأساسي للمشروع.
- تم تجهيز طبقة توثيق وتنفيذ أولية متقدمة داخل المستودع.
- `backend/` أصبح الآن مشروع Laravel فعليًا ويحتوي على `composer.json`, `artisan`, و`vendor/`.
- `frontend/` أصبح الآن مشروع React/Vite فعليًا ويحتوي على `package.json` وبنية تشغيل قابلة للبناء.
- تم فصل البوابة العامة `Portal` عن نظام المدرسة الداخلي على مستوى الواجهة والتنقل، مع الإبقاء على الوحدات الحالية كما هي داخل `/app`.
- تم تنفيذ `migrations` و`seeders` فعليًا داخل Laravel.
- تم تنفيذ الوحدة الأساسية التالية فعليًا داخل الـ API:
  - `Auth`
  - `Schools`
  - `Users`
  - `Students`
- `IEP`
- `Messages`
- `Notifications`
- `Reports`
- `Supervision`
- `Files`
- تم بناء بوابة عامة مستقلة تشمل الصفحات: `الرئيسية`, `عن القسم`, `الخدمات`, `البرامج`, `الإحصائيات`, `الخريطة التفاعلية`, `تواصل معنا`, `تسجيل الدخول`.
- تم إضافة `Portal API` عامة للبوابة تشمل البرامج والمدارس والإحصائيات بدون كشف البيانات الداخلية.
- تم إضافة نظام دخول جديد ثنائي المسار داخل نفس المشروع:
  - `دخول مدرسي` للحسابات المرتبطة بمدرسة مع التحقق من `school_code`
  - `دخول مركزي` للحسابات الإدارية والإشرافية مع التحقق من `is_central`
- تم تطبيق منطق توجيه بعد تسجيل الدخول بدون إظهار الأدوار في الواجهة:
  - مستخدم مدرسة واحدة → `/schools/{id}`
  - مشرف متعدد المدارس → `/select-school`
  - أدمن / سوبر أدمن → `/app`
- تم تجهيز Browser Demo مؤقت داخل Laravel على المسار `/` لتجربة النظام من المتصفح، ويشمل الآن تدفق `IEP` الأساسي من الإنشاء حتى الاعتماد.
- تم توسيع Browser Demo ليشمل أيضًا المراسلات الداخلية `Messages` والإشعارات `Notifications` والتقارير `Reports` والملفات `Files` على مستوى الاستعراض والتفاعل المباشر.
- تم تنفيذ `Supervision` فعليًا على مستوى الـ API مع جدولة الزيارات وإنهائها والتوصيات، ولم تُربط بعد بواجهة الـ Browser Demo.
- تم تفعيل العزل متعدد المدارس `tenant context` وربطه بمسارات الطلاب.
- تم ربط الصلاحيات بالـ `policies` داخل وحدات المدارس والمستخدمين والطلاب.
- تم اعتماد تقسيم العمل إلى:
  - `backend/` للخلفية Laravel
  - `frontend/` للواجهة React
  - `docker/` لملفات الحاويات
  - `docs/` للتوثيق التنفيذي
  - `infrastructure/` لسكريبتات النشر والدعم
